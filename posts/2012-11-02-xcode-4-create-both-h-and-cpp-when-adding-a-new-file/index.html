XCode 4: create both .h and .cpp when adding a new file

Fri Nov 2, 2012

200 Words|Read in about 1 Min
Tags: xcode  

This might be what we can call a regression from Apple. Surprisingly, XCode4 doesn’t create both .cpp and .h files at the same time when adding a new C++ file, unlike XCode3. Fixing this happens inside /Developer/Library/Xcode/Templates/File Templates/C and C++ folder, so:

<table border="0" cellspacing="0" cellpadding="0">
  <tr>
    <td>
      <div>
        1
      </div>
      
      <div>
        2
      </div>
    </td>
    
    <td>
      <div>
        <div>
          <code>cd</code> <code>/Developer/Library/Xcode/Templates/File</code><code>\ Templates</code><code>/C</code><code>\ and\ C++</code>
        </div>
        
        <div>
          <code>cp</code> <code>Header\ File.xctemplate</code><code>/___FILEBASENAME___</code><code>.h C++\ File.xctemplate/</code>
        </div>
      </div>
    </td>
  </tr>
</table>

Now to make the .cpp file include the .h header, open _\_FILEBASENAME__.cpp with any editor (vim for instance) and add the following line to it:

<table border="0" cellspacing="0" cellpadding="0">
  <tr>
    <td>
      <div>
        1
      </div>
    </td>
    
    <td>
      <div>
        <div>
          <code>#include "___FILEBASENAME___.h"</code>
        </div>
      </div>
    </td>
  </tr>
</table>

and you’re done!
