关于magento的名字转化为符合国内的习惯

Tue Mar 19, 2013

800 Words|Read in about 2 Min
Tags: magento  

安装中文包后的前台翻译很生硬，找到中文包下的Mage_Customer，用记事本打开，找到First Name，然后把后面的翻译改为自己习惯的，比如“姓氏”；再找到Last Name改为自己习惯的，比如“名字（名）”；其它要改的地方请酌情改动。

 

此时基本可以了。但我们注册用户习惯直接一个“姓名”注册，因此在这里有一个小窍门儿，打开模板页（在后台打开模板路径提示，找到模板页，这里比如 是用户注册页面下的姓名修改：frontend/base/default/template/customer/widget /name.phtml），然后把first name输入框不显示，给它设定一个默认值空格（ )，如下代码：

 

<label for=”” class=”required”>*

<input type=”hidden” id=”” name=”” value=” ” title=”” class=”input-text required-entry” />

 

此时的注册和用户面板显示就很符合中国用户的习惯了。

但用户面板多了一个空格也许不是很好看，此时我打开frontend/base/default/template/customer /account/dashboard/hello.phtml页面，修改 为

public function getLastName()

{

return Mage::getSingleton(‘customer/session’)->getCustomer()->getLastname();

}

 

到此时就已经很完美了，其它地方的有关姓名显示和注册方面的依照上面的修改就行。
