6.2 调试Doctrine生成的SQL

因为你不用手写SQL语句，Doctrine会兼容不同的数据库，并生成SQL语句。但有时，查看一下Dcotrine生成的SQL是有很大帮助的；例如，调试查询不按预期工作，在开发环境中，symfony把这些查询记录在 log/ 目录中。每一个应用和每一个环境都有一个日志文件。我们来找到这个被命名为 frontend_dev.log 的文件：

# log/frontend_dev.log
Dec 04 13:58:33 symfony {sfDoctrineLogger} executeQuery : SELECT 
j.id AS j__id, j.category_id AS j__category_id, j.type AS j__type, 
j.company AS j__company, j.logo AS j__logo, j.url AS j__url, 
j.position AS j__position, j.location AS j__location, 
j.description AS j__description, j.how_to_apply AS j__how_to_apply, 
j.token AS j__token, j.is_public AS j__is_public, 
j.is_activated AS j__is_activated, j.email AS j__email, 
j.expires_at AS j__expires_at, j.created_at AS j__created_at, 
j.updated_at AS j__updated_at FROM jobeet_job j 
WHERE j.created_at > ? (2008-11-08 01:13:35)

你可以看见Doctrine有一个 where 子句对应 created_at 字段（WHERE j.created_at > ?）。

6.4 更多关于fixtures

在浏览器中刷新Jobeet首页，在数据库中的职位都是几天前才发布的，所以首页上没有任何变化。我们更改fixtures文件来增加一个已经过期的职位：

# data/fixtures/jobs.yml
JobeetJob:
  # other jobs

  expired_job:
    JobeetCategory: programming
    company:        Sensio Labs
    position:       Web Developer
    location:       Paris, France
    description:    Lorem ipsum dolor sit amet, consectetur adipisicing elit.
    how_to_apply:   Send your resume to lorem.ipsum dolor.sit
    is_public:      true
    is_activated:   true
    created_at:     '2005-12-01 00:00:00'
    token:          job_expired
    email:          <a href="mailto:job@example.com">job@example.com</a>

正如你所看到的我们在装填文件中添加了一个职位，created_at 字段值即使是自动填充我们也可以定义个值。被定义的值会覆盖默认值。重新载入装填文件并刷新浏览器，则不会显示旧职位了：

$ php symfony doctrine:data-load

你也可以执行下面查询来确保 expires_at 字段是通过 save() 方法填充的，基于 created_at 值：

SELECT `position`, `created_at`, `expires_at` FROM `jobeet_job`;

6.5 自定义配置

在 JobeetJob::save() 方法中，对于职位过期天数我们做硬编码。最好为30天。对于应用的特定设置，symfony提供一个内置的配置文件，app.yml 文件。YAML文件可以包含一些你所要的配置：

# apps/frontend/config/app.yml
all:
  active_days: 30

在应用中，这些配置通过 sfConfig 全局类访问到的：

sfConfig::get(<span class="st0">'app_active_days'</span>)

其中一个 app_ 前缀，是因为 sfConfig 类也可以访问到symfony的配置，我们稍后会看到。

6.10 安全化职位页面

当一个职位过期时，即使你知道URL，就不可能再访问了。尝试过期职位的URL（用数据库中实际的 id 替换 id – SELECT id, token FROM jobeet_job WHERE expires_at < NOW() ）：

/frontend_dev.php/job/sensio-labs/paris-france/ID/web-developer-expired

而不是显示出职位页面，我们需要用户前进到一个404页面。但如何才可以做到这一点，正如职位是通过路由自动检索的？

# apps/frontend/config/routing.yml
job_show_user:
  url:     /job/:company_slug/:location_slug/:id/:position_slug
  class:   sfDoctrineRoute
  options:
    model: JobeetJob
    type:  object
    method_for_query: retrieveActiveJob
  param:   { module: job, action: show }
  requirements:
    id: \d+
    sf_method:
