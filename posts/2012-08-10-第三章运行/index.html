正如你前一章学到的，symfony框架是一组用PHP写下得文件。

symfony项目要用这些文件。

Symfony要求版本至少PHP 5.2.4。

检查一下你是否安装了，打开命令行输入这个命名：

$ php -v

PHP 5.3.1 (cli) (built: Jan  6 2010 20:54:10) 
Copyright (c) 1997-2009 The PHP Group
Zend Engine v2.3.0, Copyright (c) 1998-2009 Zend Technologies

如果版本数字是5.2.4或更高，然后我们马上开始本章。

3.1 先决条件

安装symfony之前，你需要检测你的电脑已正确安装和配置。

花时间认真读本章，并按照所有步骤所要求过的来检查你的配置，因为会为你继续下面的内容节省时间。

3.1.1 第三方软件

首先，你需要检查你的电脑对于web开发有一个友好的工作环境。

<span en=“At a minimum, you need a web server (Apache, for instance), a database engine (MySQL, PostgreSQL, SQLite, or any <a target="_target” href="http://www.php.net/PDO">PDO-compatible database engine), and PHP 5.2.4 or later.”>最低限度，你需要一个web服务器（例如，Apache），一个数据库引擎（MySQL、PostgreSQL、SQLite，或一些支持PDO的数据库引擎），还有PHP 5.2.4或更高。

3.1.2 命令行界面

Symfony框架自带了一个命令行工具，自动为你做一些事情。

如果你是类Unix OS用户，你会感觉到家了。

<span en=“If you run a Windows system, it will also work fine, but you will just have to type a few commands at the cmd prompt.”>如果你运行Windows系统，那同样会运行的很好，但你只是必须输入少量的命令在 cmd 提示。

3.1.3 PHP配置

每种操作系统的PHP配置可能都有所不同，甚至不同的Linux发行版之间，你需要检查你的PHP配置满足PHP最低要求。

<span en=“First, ensure that you have PHP 5.2.4 as a minimum installed by using the phpinfo() built-in function or by running php -v on the command line.”>首先，确保你运行 phpinfo() 内建函数或在命令行运行 php -v 查看版本号，满足PHP 5.2.4最低安装要求。

在一些配置上要注意，你可能安装了两个不同的PHP版本：一个对于命令行，和另外一个对WEB。

3.3.2 配置数据库

<span en=“The symfony framework supports all PDO-supported databases (MySQL, PostgreSQL, SQLite, Oracle, MSSQL, …) out of the box. On top of <a target="_blank” href="http://www.php.net/PDO">PDO, symfony comes bundled with two ORM tools: Propel and Doctrine.”>symfony框架支持所有的PDO所支持的数据库（MySQL、PostgreSQL、SQLite、Oracle、MSSQL、…）。

symfony自带了两个ORM工具：Propel和Doctrine。

当创建一个新项目，Doctrine是默认激活的。

<span en=“Configuring the database used by Doctrine is as simple as using the configure:database task:”>用 configure:database 命令来配置Doctrine所使用的数据库就是那么简单：

$ php symfony configure:database "mysql:host=localhost;dbname=dbname" root mYsEcret

<span en=“The configure:database task takes three arguments: the <a target="_blank” href="http://www.php.net/manual/en/pdo.drivers.php">PDO DSN, the username, and the password to access the database.”>configure:database 命令来访问数据库带有三个参数：PDO DSN、用户名、密码。

如果你不需要密码来访问你的数据库，则忽略第三个删除。

3.4 web服务器配置

3.4.1 丑陋的方式

如果你在你的WEB服务器的WEB根目录下某个地方创建了项目目录，你已经能在WEB浏览器中访问项目。

<span en=“Of course, as there is no configuration, it is very fast to set up, but try to access the config/databases.yml file in your browser to understand the bad consequences of such a lazy attitude.”>当然，作为没有经过任何配置的，建起来很快，但在你的浏览器中尝试访问 config/databases.yml 文件，就会了解这样一个懒惰的态度的不良后果。

如果用户知道你的网站是用symfony开发的，他可能会访问到一些敏感文件。

<span en="Never ever use this setup on a production server, and read the next section to learn how to configure your web server properly.”>永远不要在产品服务器上使用这种配置，并阅读下一节学习怎样正确的配置你的WEB服务器。

3.4.2 安全的方式

一个好的WEB实践是只把需要被WEB浏览器访问的文件放在WEB根目录下，比如CSS、JavaScript、图片。

<span en=“By default, we recommend storing these files under the web/ sub-directory of a symfony project.”>默认情况下，一个symfony项目我们建议把这些文件放在 web/ 子目录下。

<span en=“If you have a look at this directory, you will find some sub-directories for web assets (css/ and images/) and the two front controller files.”>如果你看过这个目录，你会发现一些存放WEB资源的子目录（css/ 和 images/）和两个前段控制器文件。

前段控制器仅仅是个PHP文件，需要放在WEB根目录下。

从浏览器可以隐藏所有其他PHP文件。这样做对于安全性是很有益处的。

web服务器的配置

现在是更改你的Apache服务器的时候了。

<span en=“Locate and open the httpd.conf configuration file and add the following configuration at the end:”>找到并打开 httpd.conf 配置文件并添加下面这些配置在其末尾：

<span en="# Be sure to only have this line once in your configuration"># 确保必须要有这行</span>
NameVirtualHost 127.0.0.1:8080

<span en="# This is the configuration for your project"># 这是对于你项目的配置</span>
Listen 127.0.0.1:8080

<VirtualHost 127.0.0.1:8080>
  DocumentRoot "/home/sfprojects/web"
  DirectoryIndex index.php
  <Directory "/home/sfprojects/web">
    AllowOverride All
    Allow from All
  </Directory>
  
  Alias /sf /home/sfprojects/lib/vendor/symfony/data/web/sf
  <Directory "/home/sfprojects/lib/vendor/symfony/data/web/sf">
    AllowOverride All
    Allow from All
  </Directory>
</VirtualHost>

<span en=“This configuration makes Apache listen to port 8080 on your machine, so the website will be accessible at the following URL:”>这个配置使Apache在你的机器监听 8080 端口，因此，重启Apache之后，将会用这个URL网址访问网站：

<span en=“You can change 8080 to any number, but favour numbers greater than 1024 as they do not require administrator rights.”>你可以更改 8080 为另外一些数字，但数字最好大于 1024，因为他们不需要管理员权限。
