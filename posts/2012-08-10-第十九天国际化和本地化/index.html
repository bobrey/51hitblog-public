19.5 国际化

19.5.1 语言、字符集、编码

不同的语言有不同的字符集。

英语是最简单的一个，因为它只用ASCII字符，法语更复杂一点，用了强调字符比如“é”，还有一些语言像俄语、中文、阿拉伯语就更复杂了，因为它们所有的字符都是在ASCII范围之外。

这样的语言都是用完全不同的字符集来定义的。

当处理国际化数据时，最好使用unicode标准。

unicode背后的想法是建立一个通用的字符集，包含所有语言的字符。

用unicode的问题是，一个字符就多达21个字节。

因此，对于web，我们使用UTF-8，它映射unicode代码指向可变长度的字节序列。

在UTF-8中，大多数使用的语言有它们的字符编码，不到3个字节。

<span en=“UTF-8 is the default encoding used by symfony, and it is defined in the settings.yml configuration file:”>UTF-8是symfony默认使用的编码，在 settings.yml 配置文件中定义：

<span en=“Also, to enable the internationalization layer of symfony, you must set the i18n setting to true in settings.yml:”>另外，要开启symfony的国际化层，你必须在 settings.yml 中设置 i18n 为 true：

19.5.2 模版

一个国际化网站意味着用户界面被翻译成了几种语言。

<span en=“In a template, all strings that are language dependent must be wrapped with the __() helper (notice that there is two underscores).”>在模版中，所有依赖于语言的字符串都必须用 __() 辅助函数包起来（注意是两个下划线）。

<span en=“The __() helper is part of the I18N helper group, which contains helpers that ease i18n management in templates.”>__() 辅助函数是 I18N 辅助函数组的一部分，这些包含的辅助函数以缓解模版中的i18n管理。

<span en=“As this helper group is not loaded by default, you need to either manually add it in each template with use_helper(&#8216;I18N&#8217;) as we already did for the Text helper group, or load it globally by adding it to the standard_helpers setting:”>因为这个辅助函数组不是默认加载的，你需要要么在每个模版中用 use_helper('I18N') 手动添加它，因为我们已经做了 Text 辅助函数组，要么通过添加它到 standard_helpers 设置来全局的加载它。

<span en=“Here is how to use the __() helper for the Jobeet footer:”>这是对Jobeet页脚如何使用 __() 辅助函数：

<span en=“When symfony renders a template, each time the __() helper is called, symfony looks for a translation for the current user’s culture.”>当symfony渲染模版时，每次 __() 辅助函数都要被调用，symfony为用户的当前culture查找翻译。

如果翻译被找到，则使用，如果没有，第一个参数则被返回作为一个后备值。

所有翻译都储存在一个编目里面。

i18n框架提供许多不同的策略来储存翻译。

我们会使用“XLIFF”格式，这是一个标准，非常灵活。

它还储存管理后台生成器和大多数symfony插件所使用的翻译。

<h3>
  <a name="chapter_19_sub_translations_with_arguments"><span en="19.5.4 Translations with Arguments">19.5.4 用参数翻译</span></a>
</h3>

<p>
  <span en="The main principle behind internationalization is to translate whole sentences.">国际化背后主要原则是翻译整条句子。</span><br /> <span en="But some sentences embed dynamic values.">但有些句子被嵌入了动态值。</span><br /> <span en="In Jobeet, this is the case on the homepage for the &quot;more...&quot; link:">在Jobeet中，对于首页上的“more&#8230;”链接这个情况：</span>
</p>

<pre class="php">&lt;!-- apps/frontend/modules/job/templates/indexSuccess.php --&gt;

<div class=“more_jobs”> and <?php echo link_to($count, ‘category’, $category) ?> more… </div>

<p>
  <span en="The number of jobs is a variable that must be replaced by a placeholder for translation:">职位的数目是一个变数必须由一个占位符所替换：</span>
</p>

<pre class="php">&lt;!-- apps/frontend/modules/job/templates/indexSuccess.php --&gt;

<div class=“more_jobs”> <?php echo __(‘and %count% more…', array('%count%' => link_to($count, ‘category’, $category))) ?> </div>

<p>
  <span en="The string to be translated is now "and %count% more...", and the <code>%count%</code> placeholder will be replaced by the real number at runtime, thanks to the value given as the second argument to the <code>__()</code> helper.&#8221;>要被翻译的字符串现在是“and %count% more&#8230;”，然后 <code>%count%</code> 占位符会由运行时真正的数字所替换，得意于给定的值作为<b style="color:black;background-color:#ffff66">第</b>2个参数到 <code>__()</code> 辅助函数。</span>
</p>

<p>
  <span en="Add the new string manually by inserting a <code>trans-unit</code> tag in the <code>messages.xml</code> file, or use the <code>i18n:extract</code> task to update the file automatically:&#8221;>通过在 <code>messages.xml</code> 文件中手动插入一个 <code>trans-unit</code> 标签来添加新的字符串，或者使用 <code>i18n:extract</code> 命令来自动的更新文件：</span>
</p>

<pre class="command-line">$ php symfony i18n:extract frontend fr --auto-save</pre>

<p>
  <span en="After running the task, open the XLIFF file to add the French translation:">运行命令之后，打开XLIFF文件来添加法语翻译：</span>
</p>

<pre class="xml"><span class="sc3"><span class="re1">&lt;trans-unit</span> <span class="re0">id</span>=<span class="st0">"6"</span><span class="re2">&gt;</span></span>

<source>and %count% more…</source> <target>et %count% autres…</target> </trans-unit>

<p>
  <span en="The only requirement in the translated string is to use the <code>%count%</code> placeholder somewhere.&#8221;>在翻译的字符串中唯一的要求是有些地方要使用 <code>%count%</code> 占位符。</span>
</p>

<p>
  <span en="Some other strings are even more complex as they involve plurals.">一些其他的字符串甚至更复杂，因为它们包括复数。</span><br /> <span en="According to some numbers, the sentence changes, but not necessarily the same way for all languages.">根据一些数字，改变句子，而且对于所有语言不一定都是一样的方法。</span><br /> <span en="Some languages have very complex grammar rules for plurals, like Polish or Russian.">一些语言有很复杂的复数语法规则，比如波兰语或俄语。</span>
</p>

<p>
  <span en="On the category page, the number of jobs in the current category is displayed:">在分类页上，当前分类职位的数目是要显示的：</span>
</p>

<pre class="php">&lt;!-- apps/frontend/modules/category/templates/showSuccess.php --&gt;

<strong><?php echo count($pager) ?></strong> jobs in this category

<p>
  <span en="When a sentence has different translations according to a number, the <code>format_number_choice()</code> helper should be used:&#8221;>当一个句子根据一个数字有不同的翻译时，则应该使用 <code>format_number_choice()</code>：</span>
</p>

<pre class="php"><span class="kw2">&lt;?php</span> <span class="kw3">echo</span> format_number_choice<span class="br0">&#40;</span>
<span class="st0">'[0]No job in this category|[1]One job in this category|(1,+Inf]%count% jobs in this category'</span>,
<span class="kw3">array</span><span class="br0">&#40;</span><span class="st0">'%count%'</span> =&gt; <span class="st0">'&lt;strong&gt;'</span>.<span class="kw3">count</span><span class="br0">&#40;</span><span class="re0">$pager</span><span class="br0">&#41;</span>.<span class="st0">'&lt;/strong&gt;'</span><span class="br0">&#41;</span>,
<span class="kw3">count</span><span class="br0">&#40;</span><span class="re0">$pager</span><span class="br0">&#41;</span>

) ?>

<p>
  <span en="The <code>format_number_choice()</code> helper takes three arguments:&#8221;><code>format_number_choice()</code> 辅助函数有3个参数：</span>
</p>

<ul>
  <li>
    <span en="The string to use depending on the number">字符串依赖的数字</span>
  </li>
  <li>
    <span en="An array of placeholders">占位符数组</span>
  </li>
  <li>
    <span en="The number to use to determine which text to use">用于决定要使用哪个文本的数字</span>
  </li>
</ul>

<p>
  <span en="The string that describes the different translations according to the number is formatted as follow:">根据数字被格式化为以下各项字符串描述不同的翻译：</span>
</p>

<ul>
  <li>
    <span en="Each possibility is separated by a pipe character (<code>|</code>)&#8221;>每个可能性是由一个管道字符（<code>|</code>）分隔的</span>
  </li>
  <li>
    <span en="Each string is composed of a range followed by the translation">每个字符串是一个区间所组成的，以下是区间的描述规则</span>

<p>
  <span en="The range can describe any range of numbers:">该区间可以描述任何范围的数字：</span>
</p>

<ul>
  <li>
    <span en="<code>[1,2]</code>: Accepts values between 1 and 2, inclusive&#8221;><code>[1,2]</code>：接受1和2之间的值，包含1和2</span>

  <li>

  </li>
</ul>

  <span en="Translating the string is similar to other message strings:">翻译字符串类似于其他消息字符串：</span>
</p>

  <span en="For the Jobeet website, we won't internationalize all tables as it does not make sense to ask the job posters to translate their job posts in all available languages.">对于Jobeet网站，我们不会国际化所有的表，因为要求职位的发布者用所有的语言来翻译他们的职位帖子是没有意义的。</span><br /> <span en="But the category table definitely needs to be translated.">但分类表肯定是要翻译的。</span>
</p>

<p>
  <span en="The Doctrine plugin supports i18n tables out of the box.">Doctrine插件支持i18n表。</span><br /> <span en="For each table that contains localized data, two tables need to be created: one for columns that are i18n-independent, and the other one with columns that need to be internationalized.">每个表包含了本地化的数据，需要创建两个表：其中一个表的列不依赖于i18n，然后另一个表的列需要被国际化。</span><br /> <span en="The two tables are linked by a one-to-many relationship.">两个表通过一对多关系来连接的。</span>
</p>

<p>
  <span en="Update the <code>schema.yml</code>|<code>schema.yml</code> (I18n) accordingly:&#8221;>更新 <code>schema.yml</code>：</span>
</p>

<pre># config/doctrine/schema.yml

JobeetCategory: actAs: Timestampable: ~ I18n: fields: [name] actAs: Sluggable: { fields: [name], uniqueBy: [lang, name] } columns: name: { type: string(255), notnull: true }

<p>
  <span en="By turning on the <code>I18n</code> behavior, a model named <code>JobeetCategoryTranslation</code> will be automatically created and the specified <code>fields</code> are moved to that model.&#8221;>通过打开 <code>I18n</code> 行为，一个叫做 <code>JobeetCategoryTranslation</code> 的模型会自动的被创建，然后指定的 <code>fields</code> 被移动到了那个模型。</span>
</p>

<p>
  <span en="Notice we simply turn on the <code>I18n</code> behavior and move the <code>Sluggable</code> behavior to be attached to the <code>JobeetCategoryTranslation</code> model which is automatically created.&#8221;>注意，仅仅打开 <code>I18n</code> 行为然后移动 <code>Sluggable</code> 行为来附属到 <code>JobeetCategoryTranslation</code> 模型，这是自动创建的。</span><br /> <span en="The <code>uniqueBy</code> option tells the <code>Sluggable</code> behavior which fields determine whether a slug is unique or not.&#8221;><code>uniqueBy</code> 选项告诉 <code>Sluggable</code> 行为，这些字段决定一个slug是否是唯一的。</span><br /> <span en="In this case each slug must be unique for each <code>lang</code> and <code>name</code> pair.&#8221;>在这个情况中，slug对每个 <code>lang</code> 和 <code>name</code> 这一对必须是唯一的。</span>
</p>

<p>
  <span en="And update the fixtures for categories:">然后更新分类的fixtures：</span>
</p>

<pre># data/fixtures/categories.yml

JobeetCategory: design: Translation: en: name: Design fr: name: design programming: Translation: en: name: Programming fr: name: Programmation manager: Translation: en: name: Manager fr: name: Manager administrator: Translation: en: name: Administrator fr: name: Administrateur

<p>
  <span en="We also need to override the <code>findOneBySlug()</code> method in <code>JobeetCategoryTable</code>.&#8221;>我们还需要重写 <code>JobeetCategoryTable</code> 里面的 <code>findOneBySlug()</code> 方法。</span><br /> <span en="Since Doctrine provides some magic finders for all columns in a model, we need to simply create the <code>findOneBySlug()</code> method so that we override the default magic functionality Doctrine provides.&#8221;>因为Doctrine给模型里面的所有列提供了一些魔术查找器，我们仅仅需要创建 <code>findOneBySlug()</code> 方法，以便我们重写Doctrine提供的默认的魔术功能。</span>
</p>

<p>
  <span en="We need to make a few changes so that the category is retrieved based on the english slug in the <code>JobeetCategoryTranslation</code> table.&#8221;>我们需要做一些改变，以便分类在 <code>JobeetCategoryTranslation</code> 表中是基于英语slug取得的。</span>
</p>

<pre class="php"><span class="co1">// lib/model/doctrine/JobeetCategoryTable.cass.php</span>

public function findOneBySlug($slug) { $q = $this->createQuery(‘a’) ->leftJoin(‘a.Translation t’) ->andWhere(’t.lang = ?', ‘en’) ->andWhere(’t.slug = ?', $slug); return $q->fetchOne(); }

<p>
  <span en="Rebuild the model:">重建模型：</span>
</p>

<pre class="command-line">$ php symfony doctrine:build --all --and-load --no-confirmation

$ php symfony cc

  <p>

</blockquote>

<p>
  <span en="When using the <code>I18n</code> behavior, proxies are created between the <code>JobeetCategory</code> object and the <code>JobeetCategoryTranslation</code> object so all the old functions for retrieving the category name will still work and retrieve the value for the current culture.&#8221;>当使用 <code>I18n</code> 行为，代理被创建在 <code>JobeetCategory</code> 对象和 <code>JobeetCategoryTranslation</code> 对象之间，所以对于取得分类名称的所有旧的函数仍然能工作，然后取得当前culture的值。</span>
</p>

<blockquote class="tip">
  <p>
    <span en="To reduce the number of database requests, join the <code>JobeetCategoryTranslation</code> in your queries.&#8221;>为了减少数据库请求数量，在你的查询中加入 <code>JobeetCategoryTranslation</code>。</span><br /> <span en="It will retrieve the main object and the i18n one in one query.">它会取得主要的对象和一个查询中的i18n对象。</span>
  </p>

  <p>
    <span en="The <code>WITH</code> keyword above will append a condition to the automatically added <code>ON</code> condition of the query.&#8221;>上面的 <code>WITH</code> 关键字会附加一个条件来自动的添加查询的 <code>ON</code> 条件。</span><br /> <span en="So, the <code>ON</code> condition of the join will end up being.&#8221;>所以，加入的 <code>ON</code> 条件将最终为。</span>
  </p>
  
  <pre class="sql"><span class="kw1">LEFT</span> <span class="kw1">JOIN</span> c.Translation t <span class="kw1">ON</span> c.id = t.id <span class="kw1">AND</span> t.lang = ?</pre>
</blockquote>

<p>
  <span en="As the <code>category</code> route is tied to the <code>JobeetCategory</code> model class and because the <code>slug</code> is now part of the <code>JobeetCategoryTranslation</code>, the route is not able to retrieve the <code>Category</code> object automatically.&#8221;>正如 <code>category</code> 路由关联了 <code>JobeetCategory</code> 模型类，而且因为 <code>slug</code> 现在是 <code>JobeetCategoryTranslation</code> 的一部分，该路由不能自动的取得 <code>Category</code> 对象。</span><br /> <span en="To help the routing system, let's create a method that will take care of object retrieval:">为了帮助路由系统，我们创建一个会照顾方法取得的方法：</span>
</p>

<p>
  <span en="Since we already overrode the <code>findOneBySlug()</code> let&#8217;s refactor a little bit more so these methods can be shared.&#8221;>因为我们已经重写了<code>findOneBySlug()</code>，我们重构一下，因此这些方法可以被共享。</span><br /> <span en="We'll create a new <code>findOneBySlugAndCulture()</code> and <code>doSelectForSlug()</code> methods and change the <code>findOneBySlug()</code> method to simply use the <code>findOneBySlugAndCulture()</code> method.&#8221;>我们会创建一个新的 <code>findOneBySlugAndCulture()</code> 和 <code>doSelectForSlug()</code> 方法，然后更改 <code>findOneBySlug</code> 方法来简单的使用 <code>findOneBySlugAndCulture()</code> 方法。</span>

<pre class="php"><span class="co1">// lib/model/doctrine/JobeetCategoryTable.class.php</span>

public function doSelectForSlug($parameters) { return $this->findOneBySlugAndCulture($parameters[‘slug’], $parameters[‘sf_culture’]); }   public function findOneBySlugAndCulture($slug, $culture = ‘en’) { $q = $this->createQuery(‘a’) ->leftJoin(‘a.Translation t’) ->andWhere(’t.lang = ?', $culture) ->andWhere(’t.slug = ?', $slug); return $q->fetchOne(); }   public function findOneBySlug($slug) { return $this->findOneBySlugAndCulture($slug, ‘en’); }

<p>
  <span en="Then, use the <code>method</code> option|<code>method</code> option (Routing) to tell the <code>category</code> route to use the <code>doSelectForSlug()</code> method to retrieve the object:&#8221;>然后，使用 <code>method</code> 选项（路由）来告诉 <code>category</code> 路由来使用 <code>doSelectForSlug()</code> 方法来取得对象：</span>
