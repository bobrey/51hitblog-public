<span en=“Curiously, the first tutorial that programmers follow when learning a new language or a framework is the one that displays “Hello, world!” on the screen.">很奇怪，每次学习新语言或者框架的第一个例子都是在屏幕上显示“Hello, world!”。

目前为止所有利用人工智能来实现交谈的尝试的结果都很差，所以电脑能问候整个世界这种想法实在有些古怪。

但是symfony并不比其他程序笨，证据是，你可以用symfony创建一个说“Hello, <你的名字>”的页面。

本章会告诉你如何创建一个模块，也就是一组页面的集合体。

你还将了解到如何建立一个页面，由于MVC，页面由一个行为和一个模板构成。

链接和表单是web交互的基础，你将在这一章了解如何在模板里增加他们，如何用行为处理他们。

4.1 建立模块框架

在第二章中我们介绍过，symfony把页面组织成模块。

建立页面之前，你必须先建立一个模块，也就是一个symfony能识别的目录结构的一个空壳。

symfony命令行工具能自动建立模块。

<span en=“You just need to call the generate:module task with the application name and the module name as arguments.”>你只需要用symfony命令行工具执行 generate:module 任务并传应用名与模块名这两个参数给它就可以了。

<span en=“In the previous chapter, you created a frontend application.”>在前一章里，你建立了 frontend 应用。

<span en=“To add a content module to this application, type the following commands:”>如果要在这个应用里增加一个 content 模块，只需要在命令行下输入下面的命令：

$ cd ~/myproject
$ php symfony generate:module frontend content

>> dir+      ~/myproject/apps/frontend/modules/content/actions
>> file+     ~/myproject/apps/frontend/modules/content/actions/actions.class.php
>> dir+      ~/myproject/apps/frontend/modules/content/templates
>> file+     ~/myproject/apps/frontend/modules/content/templates/indexSuccess.php
>> file+     ~/myproject/test/functional/frontend/contentActionsTest.php
>> tokens    ~/myproject/test/functional/frontend/contentActionsTest.php
>> tokens    ~/myproject/apps/frontend/modules/content/actions/actions.class.php
>> tokens    ~/myproject/apps/frontend/modules/content/templates/indexSuccess.php

<span en=“Apart from the actions/, and templates/ directories, this command created only three files.”>除了 actions/，templates/ 目录，这条命令只建立了三个文件。

<span en=“The one in the test/ folder concerns functional tests, and you don’t need to bother with it until Chapter 15.”>test/ 目录里的文件与单元测试有关，在第15章之前你都不用管它。

<span en=“The actions.class.php (shown in Listing 4-1) forwards to the default module congratulation page.”>actions.class.php（见例4-1）做了一个到默认模块的成功页面的跳转。

<span en=“The templates/indexSuccess.php file is empty.”>templates/indexSuccess.php 文件是空的。

<span en=“For each new module, symfony creates a default index action.”>symfony为每一个新模块建立一个 index 行为。

<span en=“It is composed of an action method called executeIndex and a template file called indexSuccess.php.”>它是由一个 execueIndex 的方法与一个叫 indexSuccess.php 的模板组成的。

<span en=“The meanings of the execute prefix and Success suffix will be explained in Chapters 6 and 7, respectively.”>execute 前缀与 Success 后缀的含义会在第6章与第7章中分别解释。

4.2 增加一个页面

symfony里面，页面背后的逻辑放在行为里面，表现放在模板里。

不需要逻辑的页面也需要一个空的行为。

4.2.1 增加一个行为

<span en=“The “Hello, world!” page will be accessible through a show action.”>我们需要一个通过 show 行为来访问“Hello, world!”的页面。

<span en=“To create it, just add an executeShow method to the contentActions class, as shown in Listing 4-2.”>要建立这个页面，只要在 contentActions 类里面增加一个 executeShow 方法，如例4-2。

<?php

class contentActions extends sfActions
{
  public function executeShow()
  {
  }
}

<span en=“The name of the action method is always executeXxx(), where the second part of the name is the action name with the first letter capitalized.”>行为方法的名字永远是 executeXxx()，方法名字的第二部分的第一个字母总是大写。

4.2.2 增加一个模板

行为需要一个模板来表现自己。

<span en=“A template is a file located in the templates/ directory of a module, named by the action and the action termination.”>模板是模块的 templates/ 目录里的一个文件，模板名字由行为名与行为终止组成。

<span en=“The default action termination is a “success,” so the template file to be created for the show action is to be called showSuccess.php.”>默认的行为终止是“success”也就是成功，所以 show 行为的模板名是 showSuccess.php。

理想的模板只包含显示代码，所以PHP代码越少越好。

显示“Hello, world!”的页面的模板可以如同例4-3中的那么简单。

如果需要在模板里执行一些PHP代码，你应该避免使用通常的PHP语法（如例4-4）。

相反，你应该在模板里面使用特殊的PHP语法，如例4-5所示，这样不是PHP程序员的人也能理解。

这样不仅最终生成的代码的缩进格式正确，而且可以让你把复杂的代码放在行为里面，因为只有控制语句（if, foreach, while 等）有特殊语法。
