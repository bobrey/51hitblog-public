                      Java

                    <p>
                      &nbsp;

                    <p>
                      </center><br /> n is not reliable in Java. Sometimes it produces a new line, othertimes you get a verticle bar. Thus, this is interpreter dependent. Instead, Sun provides the following advice.
                    </p>
                    
                    <p>
                      &nbsp;
                    </p>
                    
                    <blockquote>
                      <p>
                        A newLine() method is provided, which uses the platform&#8217;s own notion of line separator as defined by the system property <strong>line.separator</strong>. Not all platforms use the newline character (&#8216;n&#8217;) to terminate lines. Calling<br /> this method to terminate each output line is therefore preferred to writing a newline character directly. <a href="http://java.sun.com/j2se/1.3/docs/api/java/io/BufferedWriter.html" target="_blank">sun.com</a>
                      </p>
                    </blockquote>
                    
                    <p>
                      On the other hand, in its <a href="http://java.sun.com/docs/books/tutorial/uiswing/components/dialog.html" target="_blank">Dialog Box Tutorial</a>, Sun says
                    </p>
                    
                    <blockquote>
                      <p>
                        You can split the message over several lines by putting newline (n) characters inside the message string. For example:
                      </p>
                      
                      <pre>"Complete the sentence:n "Green eggs and...""


                    <p>
                      Who knows what to believe. I&#8217;ve seen n fail in text fields.
