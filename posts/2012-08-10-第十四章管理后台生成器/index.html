很多程序都是基于存储在数据库里的数据并提供访问这些数据的界面。

symfony能够基于propel或doctrine对象自动的来完成生成数据、处理模块等这样的重复任务。

如果正确定义了模型，symfony甚至可以自动生成整个网站管理后台。

本章介绍管理后台生成器的使用，这是用propel和doctrine插件捆绑在一起的。

它依赖于一个有完整语法的特别的配置文件，所以这一章的大部分篇幅会用来介绍后台生成器的各种用法。

[14.1 基于模型的代码生成][21]

在web应用程序程序里，数据访问操作可以归结为以下几类：

  • 新增（Creation）一条记录

    <li>
  <span en="Deletion of a record">删除（Deletion）一条记录</li> </ul> 
      
  <p>
    <span en="These operations are so common that they have a dedicated acronym: CRUD.">这些操作很常见，它们有一个专门的缩写：CRUD。</span><br /> <span en="Many pages can be reduced to one of them.">很多页面都可以简化成其中之一。</span><br /> <span en="For instance, in a forum application, the list of latest posts is a retrieve operation, and the reply to a post corresponds to a create operation.">例如，一个论坛程序，帖子最新列表就是一个取得（Retrieval）记录的过程，回帖子是一个新增（Creation）过程。</span>
  </p>

      <p>

      </p>
      
      <blockquote class="note">
        <p>

      <p>
        <span en="The administration generator creates a <code>field</code> for each column in the <code>schema.yml</code> file.&#8221;>管理后台生成器在 <code>schema.yml</code> 文件中为每个列创建一个 <code>field</code>。</span><br /> <span en="Under the <code>fields</code> key, you can modify the way each field is displayed, formatted, etc.&#8221;>在 <code>fields</code> 键下面，你可以更改每个字段是否显示还有格式化等等。</span><br /> <span en="For instance, the field settings shown in Listing 14-5 define a custom label class and input type for the <code>title</code> field, and a label and a tooltip for the <code>content</code> field.&#8221;>例如，例14-5为 <code>title</code> 字段定义一个自定义的label标签的class属性和输入类型，还有给 <code>content</code> 字段添加的label标签和tooltip提示。</span><br /> <span en="The following sections will describe in detail how each parameter works.">下面的章节会详细描述每个参数如何工作。</span>
      </p>
      
      <p class="figure">
        <span en="Listing 14-5 - Setting a Custom Label for a Column">例 14-5 &#8211; 给一个列设置一个自定义label标签</span>
      </p>
      
      <pre>config:

fields: title: label: Article Title attributes: { class: foo } content: { label: Body, help: Fill in the article body }

      <p>
        <span en="In addition to this default definition for all the views, you can override the field settings for a given view (<code>list</code>, <code>filter</code>, <code>form</code>, <code>new</code> and <code>edit</code>), as demonstrated in Listing 14-6.&#8221;>此外这个默认定义是全部所有的视图的定义，你可以覆盖一个给定的视图的字段设置（<code>list</code>、<code>filter</code>、<code>form</code>、<code>new</code>、<code>edit</code>），如例14-6所示。</span>
      </p>
      
      <p class="figure">
        <span en="Listing 14-6 - Overriding Global Settings View per View">例 14-6 &#8211; 全局设置</span>
      </p>
      
      <pre>config:

fields: title: { label: Article Title } content: { label: Body }   list: fields: title: { label: Title }   form: fields: content: { label: Body of the article }

      <p>
        <span en="This is a general principle: Any settings that are set for the whole module under the <code>fields</code> key can be overridden by view-specific areas.         </li>
        <li>
          <span en="<code>list</code> inherits <code>fields</code> fields&#8221;><code>list</code> 继承 <code>fields</code>

        <li>
          <span en="<code>filter</code> inherits from <code>fields</code>&#8220;><code>filter</code> 继承 <code>fields</code>
        </li>
      </ul>

      <p>

      <p class="figure">
        <span en="Listing 14-7 - Choosing the Fields to Display, in <code>modules/comment/config/generator.yml</code>&#8220;>例 14-7 &#8211; 选择要显示的字段，在 <code>modules/comment/config/generator.yml</code></span>
